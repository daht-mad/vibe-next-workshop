# 설계서로 스킬 만들기

> 내가 만든 설계서를 실제 작동하는 스킬로 변환하기

---

## 1. VSCode에서 본인 폴더 열기

1. VSCode 실행
2. **파일** → **폴더 열기** (또는 `Cmd + O`)
3. 본인 설계서가 있는 폴더 선택

---

## 2. Claude 익스텐션 열기

1. 좌측 사이드바에서 **Claude 아이콘** 클릭
2. 또는 아무 파일 클릭 후 우측 상단 **주황색 ✱ 표시** 클릭

---

## 3. 프로젝트 초기화

```
/init
```

---

## 4. Plan 모드 진입

**Shift + Tab** 누르기

> 입력창 옆에 "Plan" 표시가 보이면 성공

---

## 5. 스킬 생성 요청

```
@설계서.md 를 기반으로 skill-creator 스킬을 활용해서 스킬 만들기 계획 세워줘.
스킬 저장 위치는 전역 말고 이 프로젝트에 해줘.
```

### @로 파일이 안 보이면?

1. VSCode 좌측 파일 탐색기에서 설계서 파일 찾기
2. 파일 **우클릭** → **"경로 복사"**
3. 채팅창에 `@` 입력 후 붙여넣기

```
@/Users/내이름/Documents/daily-feedback/설계서.md
```

---

## 6. 계획 저장 요청

Claude가 계획을 보여주면, 저장을 요청합니다:

```
이 계획을 plans 폴더 만들고 그 하위에 YYMMDD-스킬이름-계획.md 로 저장해줘.
그리고 앞으로 플랜 세울 때마다 이곳에 저장하도록 CLAUDE.md 지침에 추가해줘.
```

> 예시: `plans/260206-daily-feedback-계획.md`

---

## 7. 계획 승인

```
응, 진행해줘
```

Claude가 아래와 같이 물어보면 **1번 선택**:

```
? How would you like to proceed?
❯ 1. Yes, clear context and auto-accept edits (shift+tab)
  2. Yes, clear context
  3. Yes, keep context and auto-accept edits (shift+tab)
  4. Yes, keep context
  5. No, keep planning
```

> 1번을 선택하면 계획대로 자동 실행됩니다.

---

## 8. 플랜 문서 읽어보기

plans 폴더에 저장된 계획 문서를 꼭 한 번 읽어보세요.

### MD 파일 프리뷰 설정 (선택)

매번 프리뷰로 보고 싶다면:

1. .md 파일 **우클릭**
2. **연결 프로그램** → **기본 편집기 구성**
3. **Markdown Preview** 선택

> 이후 md 파일 클릭하면 바로 프리뷰로 보여집니다.

---

## 9. 완료 확인

- [ ] plans 폴더와 계획 파일이 생성됐다
- [ ] CLAUDE.md에 plans 저장 지침이 추가됐다
- [ ] SKILL.md 파일이 생성됐다

---

## 10. 스킬 테스트

```
/내스킬이름
```

---

## 11. 연동 작업

아마 에어테이블, Gmail, 슬랙 등 연동해야 하는 것들이 있을 거예요.

**팁**: 일단 구현 시켜놓고, 하나씩 연동하면 됩니다.

---

## 12. 작업 마무리

한 작업이 끝나면 새 세션 시작:

```
/clear
```

---

## 작업 중 팁

| 상황 | 이렇게 하세요 |
|------|--------------|
| 모르겠을 때 | Claude에게 물어보기 |
| 같은 실수 반복할 때 | "이거 CLAUDE.md 지침에 추가해줘" |
| 작업 끝났을 때 | `/clear`로 새 세션 시작 |

---

## 문제 해결

| 문제 | 해결 |
|------|------|
| Plan 모드가 안 돼요 | Shift + Tab 다시 시도 |
| skill-creator가 없어요 | `/install skill-creator` |
| 스킬이 작동 안 해요 | "방금 만든 스킬 디버깅해줘" |
