# 강사 소개

> 워크샵 전에 읽어보시면 좋아요. 강사가 왜 AI 에이전트에 빠졌는지, 실제로 어떻게 쓰고 있는지 공유합니다.

---

## 강사는 누구?

**GPTers 커뮤니티 운영자**
- 비개발자 출신, 하지만 운영 시스템은 직접 구축
- n8n(노코드 자동화)으로 다양한 워크플로 만들어 씀
- Claude Code 만나고 나서 업무 방식이 완전히 바뀜

---

## 🔧 "n8n 같은 노코드 도구랑 뭐가 달라요?"

> n8n으로 운영 시스템을 구축해봤거든요. 그래서 차이점을 체감으로 알아요.

### 노코드 자동화의 한계

**1. 규칙 없는 데이터? 불가능.**

파트너스 시스템을 만들 때, 카톡 닉네임과 멤버DB 닉네임을 매칭해야 했어요.

| 카톡 닉네임 | 멤버DB 닉네임 |
|------------|--------------|
| 지피터스 유니님 | 유니 |
| 김다민 미또 | 김다민 |
| ... | ... |

규칙이 없어요. 어떤 건 이름만, 어떤 건 닉네임만, 어떤 건 둘 다.
n8n으로? **아예 불가능.** 어느 것 하나 정확히 일치하는 게 없으니까요.

에이전트는? **맥락으로 이해해요.** "이 두 개가 같은 사람인 것 같은데?" 하고 추론합니다.

**2. 간단한 것도 복잡해짐**

웹사이트 들어가서 내용 추출하기. 인간이 하면 30초예요.

n8n으로 자동화하려면?
```
HTTP Request → HTML 가져오기 → Parse 노드 → 추출 → Code 노드로 전처리 → ...
```
노드 4-5개는 기본. 솔직히 저도 하기 싫어요.

에이전트한테는? **"이 페이지에서 가격 정보 추출해줘"** 끝.

**3. API 연동의 고통**

n8n에서 새 API 연동할 때:
```
HTTP 노드 불러오기 → API 문서 열기 → endpoint URL 복붙 
→ 파라미터 토씨 하나 안틀리고 입력 → 실행 → 에러 
→ 에러 메시지 복사 → GPT에 붙여넣기 → 해결방안 받기 
→ 다시 n8n으로 돌아와서 수정 → 또 에러 → ...
```

Claude Code에서는? **"Airtable 연동해줘"** 한마디면 끝.
API 문서 읽고, 코드 짜고, 테스트까지 알아서 해요.

### 결국 뭐가 다른가?

| 관점 | 노코드 자동화 (n8n 등) | AI 에이전트 (Claude Code) |
|------|----------------------|-------------------------|
| **동작 방식** | 규칙 기반 (IF-THEN) | 맥락 이해 |
| **데이터 처리** | 정확히 일치해야 함 | 유사하면 추론 |
| **설계 역량** | 필수 (내가 로직 다 짜야 함) | 의도만 전달하면 됨 |
| **에러 대응** | 내가 디버깅 | AI가 알아서 수정 시도 |
| **확장성** | 노드 추가 = 복잡도 증가 | 말로 요청 = 끝 |

> 💡 n8n이 나쁜 도구는 아니에요. 정해진 규칙대로 **24시간 자동 실행**이 필요하면 n8n이 맞아요.
> 하지만 **유연하게 판단**이 필요하거나, **빠르게 만들고 싶을 때**는 에이전트가 압도적이에요.

### ⭐ 입문: `/billing` 영수증 자동화

| Before | After |
|--------|-------|
| 32개 서비스 영수증 수작업 (96분) | `/billing` 한 번에 자동 (10분) |

**문제**: 매달 32개 구독 서비스 영수증을 수작업으로 다운로드 & 정리

**해결**: Claude Code + Playwright(브라우저 자동화) + Google Drive 연동

**배울 점**:
- 반복 노가다는 AI한테
- 사람은 2FA 인증만 처리

**실제 스킬 구조:**
```
billing/
├── SKILL.md          ← 메인 지침서 (206줄)
├── references/       ← 서비스별 상세 가이드
│   └── airtable.md
└── .mcp.json         ← 브라우저 MCP 연결
```

**SKILL.md 내용 (중략):**
```markdown
---
name: billing
description: "구독 서비스 빌링 인보이스/영수증 자동 다운로드 및 Google Drive 정리..."
---

## 워크플로

### Step 1: 서비스 선택
사용자에게 처리할 서비스를 Question 도구로 확인

### Step 2: 프리플라이트
1. Chrome 브라우저 연결 확인
2. 로그인 상태 확인 → 실패 시 안내 후 중단

### Step 3: 서비스별 다운로드
reference 파일 지침에 따라 인보이스/영수증 다운로드

### Step 4: Google Drive 업로드
중복 체크 → 폴더 생성 → 파일 업로드

### Step 5: 년도 폴더로 이동
YYMMDD에서 연도 추출 → 해당 년도 폴더로 정리

### Step 6: 완료 리포트 + 정리
리포트 출력 → ~/Downloads/ 임시 파일 삭제
```

> 💡 워크플로를 6단계로 나눠서 순서대로 실행. 새 서비스 추가할 땐 `references/` 파일 하나만 추가.

### ⭐⭐ 중급: `/write-post` 사례글 자동화

| Before | After |
|--------|-------|
| 작업 기록 → 사례글 변환 (2시간+) | `/write-post` 하나로 (15분) |

**문제**: 개발 작업 기록을 사례글로 변환하는 반복 작업

**해결**: 5개 AI 도구 세션 자동 스캔 → DEVLOG 생성 → 사례글 작성까지 통합

**배울 점**:
- 처음엔 2개 스킬 체이닝 → 쓰다 보니 하나로 합침
- 복잡한 워크플로도 Phase로 나누면 관리 가능

**실제 스킬 구조:**
```
write-post/
├── SKILL.md          ← 메인 지침서 (739줄!)
├── README.md         ← 사용 설명서
├── install_mac.sh    ← Mac 설치 스크립트
└── install_win.ps1   ← Windows 설치 스크립트
```

**SKILL.md 내용 (중략):**
```markdown
---
name: write-post
description: "DEVLOG 생성부터 AI 활용 사례 게시글 작성까지 한 번에..."
---

# Phase 1: DEVLOG 생성
- 5개 AI 도구 세션 자동 스캔 (Claude Code, OpenCode, Codex CLI, Gemini CLI, Antigravity)
- 프로젝트 경로로 매칭
- 날짜별/시간순 정렬
- 기존 DEVLOG 있으면: 이어쓰기 / 새 파일 / 덮어쓰기 선택

# Phase 2: DEVLOG 확인
- "수정할 부분 있으세요?" 질문
- 표현 다듬기, 구조 변경, 내용 보강 요청 반영

# Phase 3: 사례글 작성
- 맥락 질문 (한 번에 하나씩)
  - 문제 상황 + 시작 계기
  - 작업 하이라이트
  - 인상적인 순간 / 막혔던 부분
  - 결과 임팩트
- 제목 선정 (3-4개 추천)
- 초안 작성 → 피드백 반영
```

> 💡 739줄이지만 3개 Phase로 나눠서 진행. 중간에 사용자 확인받으니까 방향 수정도 가능.

📖 [전체 사례 보기](https://www.gpters.org/dev/post/automation-you-who-dont-yDQuH03ClRX8Oud)

### ⭐⭐ 중급: 상세페이지 섹션별 수정

| Before | After |
|--------|-------|
| 전체 파일 던지면 엉뚱한 곳도 수정됨 | 섹션별로 "이 부분만" 지시 |

**문제**: AI에게 전체 파일을 던지면 의도하지 않은 곳도 수정

**해결**: 페이지를 섹션으로 분리 → 섹션별 수정 → 사람 확인 → 다음 섹션

**배울 점**:
- 스킬 = 실행 순서를 정하는 것
- AI가 할 일 vs 사람이 확인할 일 구분

### ⭐⭐⭐ 고급: 파트너스 시스템 구축

| Before | After |
|--------|-------|
| "언젠간 해야지" 1년 반 방치 | 하루 만에 시스템 구축 |

**문제**: 커뮤니티 파트너스 시스템 필요한데 직접 개발하기엔 복잡

**해결**: Claude Code에게 전체 시스템 설계 위임. 80% 해결되면 OK.

**배울 점**:
- 1회성 복잡한 작업도 AI에게
- 완벽하지 않아도 0%보다 나음
- AI가 뼈대 만들면 사람이 다듬기 쉬움

### ⭐⭐⭐ 고급: 멀티 도구 세션 통합

| Before | After |
|--------|-------|
| 여러 AI 도구 대화가 흩어져 있음 | 하나로 통합 + 자동 분석 |

**문제**: Claude, Cursor, Copilot 등 여러 도구 대화가 흩어져 맥락 파악 어려움

**해결**: 여러 도구 세션을 하나로 통합하는 스킬 개발

**배울 점**:
- 스킬은 진화한다 (쓰다 보면 개선점 보임)
- Progressive Disclosure: 처음엔 요약만, 필요하면 상세 정보
- 자동화 도구를 자동화하는 것도 가능
