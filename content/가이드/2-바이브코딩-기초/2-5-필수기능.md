# Claude Code 필수 기능 4가지

> 알아두면 작업 효율 3배! 바이브코딩의 핵심 기능 4가지를 빠르게 익힙니다.

---

## 1. ⏪ Checkpoints & Rewind

> **"실수했어도 괜찮아요. 되돌리면 됩니다."**

### 이게 뭔가요?

Claude가 파일을 수정할 때마다 **자동으로 체크포인트**가 생깁니다.
게임의 세이브 포인트처럼, 언제든 이전 상태로 되돌릴 수 있어요.

```
작업 1 → 💾 체크포인트 1
작업 2 → 💾 체크포인트 2  
작업 3 → 💾 체크포인트 3 ← 지금 여기
          ↓
      "어, 이거 아닌데?" → Esc Esc → 체크포인트 2로 복원!
```

### 언제 쓰나요?

| 상황 | 해결 |
|------|------|
| Claude가 파일을 이상하게 수정했을 때 | `Esc` 두 번 |
| "아까 상태가 더 좋았는데..." | `Esc` 두 번 |
| 실험해보다가 원래대로 돌리고 싶을 때 | `Esc` 두 번 |

### 실습: 되돌리기 체험

**Step 1**: Claude에게 아무 파일이나 수정 요청
```
CLAUDE.md에 "테스트 문장입니다" 추가해줘
```

**Step 2**: 수정된 걸 확인 후, 되돌리기
- 키보드에서 `Esc` 키를 **빠르게 두 번** 누르세요
- 또는 패널 상단의 **되돌리기 버튼(↩️)** 클릭

**Step 3**: 파일이 원래대로 돌아왔는지 확인

> 💡 **팁**: 비개발자에게 이 기능은 **생명줄**입니다!
> 실수가 두렵지 않아요. 언제든 되돌릴 수 있으니까.

**완료 체크**:
- [ ] `Esc` 두 번으로 되돌리기 성공

## 2. 🔐 Permission Modes

> **"Claude의 자율성을 조절하세요."**

### 세 가지 모드

`Shift + Tab`을 누르면 모드가 바뀝니다.

| 모드 | 설명 | 언제 쓰나요? |
|------|------|-------------|
| **Ask before edit** (기본) | 모든 작업에 허락 받음 | 처음 배울 때, 중요한 파일 다룰 때 |
| **Edit automatically** | 파일 수정 자동 허용 | 익숙해진 후, 빠른 작업 원할 때 |
| **Plan mode** | 계획만 세우고 실행 안 함 | 큰 작업 전 미리보기 |

```
Ask before edit:     "이 파일 수정해도 될까요?" → [허용] 클릭
Edit automatically:  바로 수정 (물어보지 않음)
Plan mode:           "이렇게 할 거예요" (실행은 안 함)
```

### 실습: 모드 확인하기

1. Claude 패널 하단 확인 → 현재 모드가 보입니다
2. `Shift + Tab` 눌러서 모드 바꿔보기
3. 다시 **Ask before edit**로 돌아오기

> 💡 **추천**: 오늘 워크샵에서는 **Ask before edit** 유지하세요!
> 익숙해지면 Edit automatically로 바꾸면 됩니다.

## 3. 📎 @ 파일 참조

> **"이 파일 봐!" 라고 직접 알려주세요.**

### 사용법

채팅창에 `@`를 입력하면 **파일 목록**이 나타납니다.

```
@CLAUDE.md 이 파일 내용 요약해줘
@설계서.md 기반으로 코드 짜줘
@에러로그.txt 이 에러 해결해줘
```

### 언제 쓰나요?

| 상황 | 예시 |
|------|------|
| 특정 파일을 Claude에게 보여줄 때 | `@보고서.md 이거 분석해줘` |
| 여러 파일을 참조할 때 | `@A.md 와 @B.md 비교해줘` |
| 파일 내용 기반으로 작업할 때 | `@설계서.md 대로 구현해줘` |

### 실습: 파일 참조해보기

채팅창에 입력:
```
@CLAUDE.md 이 파일 내용을 한 줄로 요약해줘
```

> 💡 **팁**: Claude가 "무슨 파일이요?" 하고 물을 때
> `@파일명`으로 정확히 알려주면 확실합니다.

## 4. ⌨️ Slash Commands

> **"명령어 메뉴를 활용하세요."**

### 주요 명령어

채팅창에 `/`를 입력하면 **명령어 목록**이 나타납니다.

| 명령어 | 기능 | 언제 쓰나요? |
|--------|------|-------------|
| `/init` | CLAUDE.md 생성/업데이트 | 프로젝트 시작할 때 (이미 해봤죠!) |
| `/status` | 계정·버전·모델 확인 | 누구로 로그인했는지, 작업량 확인 |
| `/login` | 계정 전환 | 다른 계정으로 바꿀 때 |
| `/resume` | 이전 대화 불러오기 | 로그인 후 기존 작업 이어갈 때 |
| `/clear` | 대화 **완전히** 지우기 | 완전히 다른 주제 시작할 때 |
| `/compact` | 대화 **요약**해서 줄이기 | 대화 이어가면서 토큰 절약할 때 |
| `/help` | 도움말 보기 | 뭘 할 수 있는지 모를 때 |
| `/model` | AI 모델 변경 | 다른 모델 써보고 싶을 때 |
| `/context` | 토큰 사용량 시각화 | 얼마나 썼는지 확인할 때 |

---

### 📊 `/status` - 현재 상태 확인

```
/status
```

**보여주는 정보:**
- 현재 **로그인된 계정** (누구로 접속 중인지)
- Claude Code **버전**
- 현재 사용 중인 **AI 모델** (Sonnet, Opus 등)
- 연결 상태

> 💡 **팁**: "내가 어떤 계정으로 로그인했지?" 싶을 때 `/status` 입력!

---

### 🔄 `/login` - 계정 전환

```
/login
```

**하는 일:**
- 현재 계정에서 로그아웃
- 다른 계정으로 로그인

> 💡 **언제 계정을 바꾸나요?**
> - 개인 계정 ↔ 회사 계정 전환할 때
> - 다른 플랜(무료/유료)으로 바꿀 때

---

### 🔄 `/resume` - 이전 대화 불러오기

```
/resume
```

**하는 일:**
- 이전에 하던 대화 목록을 보여줌
- 원하는 대화를 선택해서 이어서 작업

**언제 쓰나요?**
- 새로 로그인한 후 기존 작업 이어가고 싶을 때
- 터미널을 껐다 켜서 대화가 사라졌을 때
- 어제 하던 작업을 오늘 계속하고 싶을 때

```
예시 흐름:
1. /resume 입력
2. 이전 대화 목록 표시됨
3. 이어가고 싶은 대화 선택
4. 기존 맥락 그대로 작업 계속!
```

> 💡 **팁**: 로그인 새로 하면 대화창이 비어 보이지만, `/resume`으로 기존 대화를 불러올 수 있어요!

### 🧹 `/clear` vs `/compact` - 뭐가 다른가요?

| | `/clear` | `/compact` |
|---|----------|------------|
| **하는 일** | 대화 **완전 삭제** | 대화 **요약해서 압축** |
| **맥락** | ❌ 다 사라짐 | ✅ 요약본 유지 |
| **토큰** | 0으로 리셋 | 크게 줄어듦 |
| **언제 쓰나** | 완전히 새 주제 | 같은 작업 계속할 때 |

**비유로 이해하기:**
```
/clear   = 노트 찢어서 버리기 📄🗑️ (백지 상태)
/compact = 노트 요약해서 정리하기 📝 (핵심만 남김)
```

**`/compact` 사용 예시:**
```
/compact                              # 기본 압축
/compact 코드 변경사항 위주로 기억해줘    # 특정 내용 강조
```

> 🎯 **추천**:
> - 대화 이어갈 거면 → `/compact`
> - 완전히 새로 시작할 거면 → `/clear`

---

### 📈 `/context` - 토큰 사용량 확인

```
/context
```

컬러 그리드로 **현재 토큰 사용량**을 보여줍니다.
- 초록색 = 여유 있음
- 노란색 = 좀 찼음
- 빨간색 = 거의 다 참

> 💡 빨간색이 많으면 `/compact`나 `/clear` 할 타이밍!

---

### 실습: 명령어 써보기

**Step 1**: `/status` 입력
- 현재 로그인된 계정 확인
- 어떤 모델 쓰고 있는지 확인

**Step 2**: `/context` 입력
- 토큰 사용량 시각화 확인

**Step 3**: `/compact` 입력
- 대화가 요약됨 (맥락은 유지!)

**Step 4**: `/clear` 입력 (마지막에)
- 대화가 완전히 지워짐
- CLAUDE.md 맥락은 그대로 유지됩니다

**완료 체크**:
- [ ] `/status`로 계정 확인
- [ ] `/context`로 토큰 사용량 확인
- [ ] `/compact`와 `/clear` 차이 이해

## 필수 기능 정리

| 기능 | 단축키/방법 | 핵심 |
|------|------------|------|

| **파일 참조** | `@파일명` | 특정 파일 지정 |
| **명령어** | `/명령어` | 빠른 기능 실행 |

> 🎯 **오늘 꼭 기억할 것**: `Esc Esc`로 되돌리기!
