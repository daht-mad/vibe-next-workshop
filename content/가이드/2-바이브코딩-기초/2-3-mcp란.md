# 2.3 MCP란?

> Model Context Protocol - AI용 USB-C 포트

---

## MCP = Claude가 외부 세계와 소통하는 표준 방식

쉽게 말해 **"AI용 USB-C 포트"**입니다.
USB-C 하나로 충전기, 모니터, 키보드 다 연결하듯이,
MCP 하나로 Slack, Airtable, 브라우저 등 다양한 서비스에 연결합니다.

```
┌─────────────┐         ┌─────────────┐
│             │   MCP   │   Lark      │
│   Claude    │ ◄─────► │   Airtable  │
│             │         │   브라우저   │
└─────────────┘         └─────────────┘
    여러분                외부 서비스
```

---

## MCP 안에는 뭐가 있나요?

각 MCP 서버 안에는 **Tools(도구)**가 들어있습니다.

| MCP 서버 | 안에 있는 Tools 예시 |
|----------|---------------------|
| **Lark MCP** 🏢 | 아래 상세 참고 |
| Playwright MCP | 페이지 열기, 클릭하기, 스크린샷 찍기, 텍스트 입력하기 |
| Context7 MCP | 라이브러리 문서 검색, 코드 예시 찾기 |
| Airtable MCP | 레코드 조회, 레코드 추가, 테이블 수정 |

> 🏢 **Lark MCP 상세** (우리 팀 사내 툴!)
> 
> | 카테고리 | 주요 Tools |
> |---------|-----------|
> | **메신저 (IM)** | 메시지 보내기, 채팅방 생성, 대화 기록 조회, 메시지 전달/답장, 긴급 알림 |
> | **캘린더** | 일정 목록 조회, 일정 검색, 일정 수정, 회의 그룹 생성, 회의록 생성 |
> | **문서 (Docs)** | 문서 가져오기, 협업자 추가/삭제, 권한 조회, 소유자 이전 |
> | **시트 (Bitable)** | 테이블 조회, 레코드 생성/수정/삭제 (최대 500개), 뷰 관리 |
> | **위키** | 위키 검색, 노드 조회 |
> | **연락처** | 사용자 정보 조회 (이메일/전화번호로 검색) |
> 
> Claude가 "Lark에서 오늘 회의 일정 알려줘"라고 하면 → `calendar.v4.calendarEvent.list` Tool 자동 호출!

> 💡 **비유**: MCP는 "앱", Tools는 앱 안의 "기능 버튼"이라고 생각하세요.

---

## 언제 호출되나요?

**Claude가 대화 중에 "이건 내 힘만으로 안 되겠다" 싶을 때 자동으로 호출합니다.**

```
여러분: "네이버 열어서 오늘 날씨 알려줘"
         ↓
Claude 생각: "브라우저를 열어야 하네... Playwright MCP 써야겠다"
         ↓
Claude: Playwright MCP의 "페이지 열기" Tool 호출
         ↓
결과: 브라우저가 열리고 날씨 정보를 가져옴
```

| 여러분이 하는 말 | Claude가 판단 | 호출하는 Tool |
|-----------------|--------------|---------------|
| "Lark에서 오늘 일정 알려줘" | 캘린더 필요 → Lark MCP | `get_calendar_events` |
| "Lark 시트에서 이번 주 데이터 가져와" | 시트 필요 → Lark MCP | `read_spreadsheet` |
| "이 사이트 열어서 스크린샷 찍어줘" | 브라우저 필요 → Playwright | `take_screenshot` |
| "Next.js 공식 문서에서 찾아봐" | 문서 검색 필요 → Context7 | `query-docs` |

> 🎯 **핵심**: 여러분은 **자연어로 요청**만 하면 됩니다. 
> 어떤 MCP의 어떤 Tool을 쓸지는 Claude가 알아서 판단합니다.

---

## 어떻게 작동하나요?

```
여러분 요청 → Claude 판단 → MCP 호출 → 외부 서비스 → 결과 반환
```

1. **설치**: MCP 서버를 컴퓨터에 설치 (한 번만)
2. **인식**: Claude가 설치된 MCP 목록을 자동으로 인식
3. **사용**: 대화 중 필요하면 Claude가 알아서 호출
4. **결과**: 외부 서비스 데이터를 가져오거나 작업 수행

> 💡 **비개발자 관점**: MCP 내부 작동 방식은 몰라도 됩니다. **"설치하면 Claude가 알아서 쓴다"**만 기억하세요.
