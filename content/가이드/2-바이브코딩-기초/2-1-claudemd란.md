# 2.1 CLAUDE.md란?

> Claude의 "뇌"를 이해합니다.

---

## 🧠 Claude의 "뇌"라고 생각하세요

**CLAUDE.md = 이 프로젝트에서 Claude가 기억하는 모든 것**

사람도 새 프로젝트 들어가면 온보딩 문서 받죠?
- "우리 팀은 이렇게 일해요"
- "이 프로젝트는 이런 거예요"
- "이건 주의해야 해요"

Claude도 똑같습니다. **CLAUDE.md에 적어두면 Claude가 매번 기억합니다.**

---

## 언제 읽히나요?

Claude가 대화를 시작할 때 **자동으로** 읽습니다.

```
폴더 열기 → Claude 대화 시작 → CLAUDE.md 자동으로 읽음 → 맥락 파악 완료
```

그래서 매번 "한국어로 해줘", "나는 PM이야" 같은 말을 반복할 필요가 없어요.

---

## 어떤 내용이 주로 관리되나요?

| 카테고리 | 예시 |
|---------|------|
| **프로젝트 설명** | "이 프로젝트는 고객 피드백을 분석하는 도구야" |
| **작업 스타일** | "한국어로 대화해줘", "이모지 써도 돼", "코드보다 기획 관점으로 설명해줘" |
| **주의사항** | "파일 삭제 전에 꼭 확인받아", "테스트 없이 배포하지 마" |
| **자주 쓰는 명령어** | "npm run dev로 실행", "pytest로 테스트" |
| **과거 실수 기록** | "저번에 A 파일 잘못 삭제했으니 조심해" |
| **팀 규칙** | "커밋 메시지는 한국어로", "PR은 꼭 리뷰 받고 머지" |

---

## 실제 예시

```markdown
# 프로젝트: 고객 피드백 분석 스킬

## 설명
사용자 인터뷰 녹취록을 분석해서 인사이트를 뽑는 자동화 스킬

## 작업 스타일
- 한국어로 대화
- 전문 용어는 쉽게 풀어서 설명
- 이모지 사용 OK

## 주의사항
- 원본 녹취록 파일은 절대 수정하지 마
- 분석 결과는 항상 /output 폴더에 저장

## 과거 실수
- 2024-01-15: 녹취록 파일 직접 수정해서 원본 날림 → 항상 복사본으로 작업할 것
```

---

## CLAUDE.md의 장점

| 장점 | 설명 |
|------|------|
| 반복 제거 | 매번 "한국어로 해줘" 안 해도 됨 |
| 맥락 유지 | 프로젝트 설명 반복 불필요 |
| 실수 방지 | 주의사항 적어두면 같은 실수 안 함 |
| 팀 공유 | git에 올리면 팀원 모두 같은 맥락 유지 |

---

## ⚠️ 중요: 직접 수정하지 마세요!

CLAUDE.md는 **Claude에게 업데이트 요청**하는 게 좋습니다.

**왜요?**
- 직접 수정하면 형식이 깨지거나 중복이 생길 수 있음
- Claude가 알아서 기존 내용과 잘 합쳐줌

**이렇게 요청하세요:**
```
CLAUDE.md에 "항상 한국어로 대화" 규칙 추가해줘
```

```
이 프로젝트에서 꼭 기억해야 할 것: 
테스트 파일은 절대 삭제하지 마. 
이거 CLAUDE.md에 추가해줘.
```

---

## 💡 핵심 습관: 실수하면 지침으로 남기기

작업하다가 에러가 나거나 실수를 했다면?

**Step 1**: 해결해달라고 요청
```
이 에러 해결해줘
```

**Step 2**: 해결 후, 지침 추가 요청
```
왜 이런 에러가 났어? 
다시는 같은 실수 반복하지 않도록 CLAUDE.md에 지침 추가해줘.
```

이렇게 하면 **Claude가 같은 실수를 반복하지 않습니다.**
