# 2.2 실습: /init으로 CLAUDE.md 만들기

> ⭐ 핵심 실습

---

## Step 1: Claude 익스텐션 패널 열기

1. 이미 열려 있는 **Claude 익스텐션 패널** 확인
2. 채팅 입력창이 보이면 성공

> 💡 **팁**: 안 보이면 `Cmd+Shift+P` (Mac) 또는 `Ctrl+Shift+P` (Windows) 누르고 "Claude" 검색

---

## Step 2: /init 입력하기

채팅창에 아래 내용을 입력하고 Enter:

```
/init
```

- Claude가 **사전설문 폴더를 분석**하기 시작합니다
- 스킬 설계서 등 기존 파일을 읽고 맥락을 파악합니다
- 잠시 기다리면 CLAUDE.md 초안이 나타납니다

---

## Step 3: 생성 승인하기

1. Claude가 제안한 내용을 훑어보세요
2. **"적용"** 또는 **"Accept"** 버튼 클릭
3. 파일이 생성됩니다

---

## Step 4: 생성된 CLAUDE.md 열어서 읽어보기

1. VSCode 왼쪽 **파일 탐색기**에서 `CLAUDE.md` 찾기
2. **클릭해서 파일 열기**
3. 내용을 처음부터 끝까지 **한 번 읽어보세요**

> 💡 **뭐가 적혀있나요?**
> 
> Claude가 사전설문 폴더의 **설계서**와 **설문 응답**을 분석해서
> 이 프로젝트가 뭔지, 어떤 맥락인지 정리해뒀습니다.
> 
> "아, Claude가 이렇게 이해했구나" 확인해보세요.

---

## Step 5: Claude에게 내 스타일 추가 요청하기

> ⚠️ **중요**: 파일을 직접 수정하지 말고, Claude에게 요청하세요!

채팅창에 입력:

```
CLAUDE.md에 아래 내용 추가해줘:
- 저는 [본인 직무]라서 코드보다 기획 관점 설명이 좋아요
- 항상 한국어로 대화해줘
- 이모지 사용 OK
```

Claude가 파일을 수정하면 **"적용"** 클릭

> 💡 **왜 직접 수정하면 안 되나요?**
> 
> 직접 수정하면 형식이 깨지거나 중복이 생길 수 있어요.
> Claude에게 요청하면 기존 내용과 자연스럽게 합쳐줍니다.

---

## 완료 체크

- [ ] CLAUDE.md 파일이 프로젝트 폴더에 생성됨
- [ ] Claude에게 요청해서 내 스타일을 추가함

> **막히면**: 강사에게 손 들어 질문하세요

---

## 💡 앞으로의 습관

프로젝트 작업 중에 **언제든 /init 다시 실행 가능**합니다.

| 상황 | 하면 좋은 것 |
|------|-------------|
| 파일이 많이 추가됐을 때 | `/init` 다시 실행 |
| 프로젝트 방향이 바뀌었을 때 | `/init` 다시 실행 |
| Claude가 맥락을 자꾸 잊는 것 같을 때 | "CLAUDE.md 업데이트해줘" 요청 |
| 에러가 났다가 해결됐을 때 | "이 실수 안 반복하게 CLAUDE.md에 추가해줘" |

> 🎯 **핵심**: CLAUDE.md는 한 번 만들고 끝이 아닙니다.
> 프로젝트하면서 **계속 업데이트**하는 거예요.
