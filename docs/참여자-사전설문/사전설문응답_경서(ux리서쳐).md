# 사전설문 응답 기록

**스킬명**: virtual-user-interviewer
**일시**: 2026-02-05

## Q1. 지금 회사에서 어떤 일을 맡고 계세요?
**A1.** UX리서처. 1. AI툴을 통해 가상의 사용자 패널을 관리 2. 가상의 인터뷰 수행 3. 인터뷰 결과를 분석 4. 다수의 인터뷰내용에서 공통 인사이트를 도출 5. 프로토타입된 이미지를 기반으로 가상의 인터뷰와 반응조사 수행

## Q2. 이 5단계 작업할 때 하루가 어떻게 흘러가세요? 어떤 AI 툴을 주로 쓰시는지, 각 단계에서 어디에 정보를 기록하는지 궁금해요!
**A2.** 지금은 GPT를 주로 사용

## Q3. GPT로 가상 사용자 패널을 만들 때, 구체적으로 어떻게 하세요?
**A3.** 100여명 정도, 다른 프로파일 특성을 가진 사용자를 생성하여 보유하고, 이중 조사의 특성에 따라 필요한 사람을 유저아이디로 도출하여 인터뷰에 적용하고 있어

## Q4. 이 100명의 프로파일 정보는 어디에 저장하고 관리하세요?
**A4.** 지금은 엑셀인데 추후 구글시트로 통합예정

## Q5. 조사에 맞는 사람을 유저 ID로 선별한 다음에는 어떻게 하세요?
**A5.** 프로파일을 한줄로 만들어(GPT가 헷갈리지 않게) 복붙하고있어

## Q6. 인터뷰는 한 번에 몇 명씩 진행하세요?
**A6.** 한번에 한명씩

## Q7. 그럼 각 사람과의 인터뷰 결과는 어디에 저장하세요?
**A7.** 인터뷰 한건이 끝나면 그사람에 해당하는 구글 닥스에 저장

## Q8. 구글 닥스는 사람마다 문서가 따로 있는 건가요, 아니면 한 프로젝트에 모든 인터뷰를 모아두는 건가요?
**A8.** 지금은 수동으로, 한명에 하나의 파일로 저장중. 추후에는 다양한 사람이더라도 하나의 파일에서 통합으로 관리하고싶음

## Q9. 인터뷰가 다 끝나고 나면, 공통 인사이트를 도출하기 위해 뭘 하세요?
**A9.** GPT에 다시 붙여넣어서, 공통 페인포인트와 니즈, 인사이트를 도출해달라고 요청

## Q10. GPT가 도출해준 공통 인사이트는 어디에 저장하거나 공유하세요?
**A10.** 새로운 구글 닥스

## Q11. 프로토타입 이미지 기반 반응조사는 어떻게 하세요?
**A11.** 이거어때요는 아니고, 다양한 질문을 함께 입력

## Q12. 프로토타입 반응조사 결과도 구글 닥스에 저장하나요?
**A12.** 맞음

## Q13. 이 전체 과정 중에서 "아 이거 왜 매번 해야 해..." 싶은 부분이 어디예요?
**A13.** 각각의 단계를 매번 명령을 해야하는것 To cgpt

## Q14. 자주 반복하는 명령어가 있나요?
**A14.** 맞음

## Q15. 혹시 자주 쓰는 프롬프트 템플릿이 있으세요?
**A15.** 구글닥스에서 질문지 퀘스처네이어를 관리함

## Q16. 최종 분석 보고서에는 어떤 내용이 꼭 들어가야 하나요?
**A16.** 일단은 워크샵이니까 3개만 하면 충분해 (공통 페인포인트, 니즈, 인사이트)

## Q17. 마지막으로, 결과물을 구글 닥스에 저장한 다음에는 어떻게 하세요?
**A17.** 라크에 공유

## Q18. 라크에 공유할 때는 어떻게 하세요?
**A18.** 구글 링크로 전송

## Q19. 이 중에서 제일 먼저 자동화하고 싶은 게 뭐예요?
**A19.** 인터뷰 자동화: 패널 선별 → 자동 인터뷰 → 결과 저장
