# 사전설문 응답 기록

**스킬명**: prd-writer
**일시**: 2026-02-05

---

## Q1. 지금 회사에서 어떤 일을 맡고 계세요?
**A1.** PM으로 일하고 있고, PRD 작성을 많이 해요.

## Q2. PRD 작성할 때 하루가 어떻게 흘러가세요? 어디서 정보를 모으고, 어떤 툴로 문서를 쓰시는지?
**A2.** 회의나 라크 메신저를 통해서 내부 구성원의 의견이나 니즈를 파악하고, cursor에 markdown문서로 작성한뒤, 다시 lark 문서로 작성하여 공유해요. 그리고 그에 맞춰서 jira티켓을 작성해요.

## Q3. 이 중에서 "아 이거 왜 매번 해야 해..." 싶은 부분이 어디예요?
**A3.** 그냥 PRD 쓰는것 자체가 제일 귀찮아요. 특히 PRD를 AI로 작성후에 리뷰하는 것도 귀찮아요.

## Q4. AI가 써준 걸 리뷰할 때 뭐가 제일 걸리세요?
**A4.** 쓸데없는 내용이 너무 많아서 지워요. 뜬구름은 아니에요.

## Q5. 혹시 회사에서 쓰는 PRD 템플릿이나 포맷이 있으세요?
**A5.** 템플릿 있어요. (SKILL.md 및 docx 템플릿 공유)

## Q6. 어떤 섹션이 필요 없는 경우가 많아요?
**A6.** "비기능 요구사항", "성공지표", "개발 일정" 등이 잘 안쓰여요. 근데 아주 가끔은 있었으면 좋겠어요.

## Q7. PRD 작성을 시작할 때 어떤 정보로 시작하세요?
**A7.** 요청 받은 내용이랑 제 머리속 내용이랑 현재 서비스의 페이지를 보고 써요.

## Q8. "현재 서비스 페이지"는 어떻게 참고하세요?
**A8.** 직접 보거나 cursor에서 해당 폴더를 보거나 mobbin이라는 서비스에서 타사 서비스를 보거나 해요.

## Q9. 최종 결과물은 Cursor에서 markdown 파일로 끝나면 되는 건가요? 아니면 Lark 문서로 자동으로 올라가면 더 좋겠어요?
**A9.** 문서에서 자동으로 올라가면 좋아요. jira 티켓은 별도 workflow로 분리하는게 유리해요.

## Q10. Lark에 문서 올릴 때, 특정 폴더나 위치가 있어요?
**A10.** 네, PRD 폴더에 올려요.

## 요약

- **핵심 페인포인트**: AI가 PRD를 써주면 쓸데없는 내용이 너무 많아서 지우는 시간이 듦
- **원하는 결과**: 회사 템플릿에 맞게, 필요한 섹션만 선별해서 PRD 작성 + Lark 자동 업로드
- **스코프 아웃**: Jira 티켓 생성 (별도 워크플로우)
- **외부 연동**: Lark (문서 업로드)
